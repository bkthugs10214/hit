# Architecture Decision Records

Each ADR captures a design choice that future-you (or a teammate) might otherwise look at and ask "wait, why did we do it this way?"

ADRs are **immutable once Accepted.** To change a decision:

1. Write a new ADR with `Status: Supersedes NNNN`.
2. Edit the old ADR's status line to `Superseded by MMMM`. (This is the only legal post-accept edit.)

Don't write ADRs for tactical fixes, library version bumps, or formatting/style choices — those belong in the journal or the commit message.
